# Runtime Directory (`.anatoly/`)

Anatoly stores all runtime artifacts ‚Äî reviews, reports, caches, vector embeddings, and lock state ‚Äî in a `.anatoly/` directory at the project root. This directory is gitignored and entirely ephemeral: it can be safely deleted at any time and will be rebuilt on the next run.

## Run Directories

Each `anatoly run` execution creates a timestamped directory under `.anatoly/runs/`.

**Source:** `src/utils/run-id.ts`

### Run ID Format

```
YYYY-MM-DD_HHmmss
```

Example: `2026-03-15_142345`

Generated by `generateRunId()`. Custom IDs can be provided with `--run-id` and must match `[a-zA-Z0-9_-]{1,64}`.

### Creation

`createRunDir()` creates the directory structure:

```
.anatoly/runs/<runId>/
‚îú‚îÄ‚îÄ logs/       # created immediately
‚îî‚îÄ‚îÄ reviews/    # created immediately
```

Then updates the `latest` pointer.

### Latest Pointer

`.anatoly/runs/latest` is a **symlink** pointing to the most recent run ID. On Windows (where symlinks may require elevated privileges), it falls back to a plain text file containing the run ID.

Used by `report` and other commands to resolve the default run when no `--run` flag is provided.

### Run Contents

| File | Created By | Description |
|------|------------|-------------|
| `reviews/*.rev.json` | review-writer | Machine-readable verdict per file (Zod-validated) |
| `reviews/*.rev.md` | review-writer | Human-readable review per file |
| `logs/*.log` | run command | Transcript of axis evaluations per file |
| `report.md` | reporter | Report index with executive summary and shard links |
| `report.N.md` | reporter | Shard reports (up to 10 files each) |
| `run-metrics.json` | run command | Execution summary: duration, cost, findings |
| `anatoly.ndjson` | pino logger | Structured event log (one JSON object per line) |

## Task Cache

**Location:** `.anatoly/tasks/*.task.json`

One file per scanned source file, containing AST-derived metadata:

- File path and content hash
- Symbol list (functions, methods, hooks, classes, types, etc.)
- Import/export maps
- Coverage data (if available)

Task files are regenerated on each scan. The content hash enables incremental review ‚Äî unchanged files can be skipped.

**Naming:** `src/utils/format.ts` ‚Üí `.anatoly/tasks/src-utils-format.task.json`

## Progress Tracking

**Location:** `.anatoly/cache/progress.json`

**Source:** `src/core/progress-manager.ts`, `src/utils/cache.ts`

Tracks the review state of each file across the current run:

```json
{
  "files": {
    "src/cli.ts": {
      "status": "DONE",
      "hash": "a1b2c3..."
    }
  }
}
```

### File Status Lifecycle

```
PENDING ‚Üí IN_PROGRESS ‚Üí DONE
                      ‚Üí ERROR
                      ‚Üí TIMEOUT
                      ‚Üí CACHED
```

| Status | Meaning |
|--------|---------|
| `PENDING` | Not yet reviewed |
| `IN_PROGRESS` | Currently being evaluated |
| `DONE` | Review complete |
| `ERROR` | Evaluation failed (will retry on next run) |
| `TIMEOUT` | Evaluation timed out |
| `CACHED` | Skipped ‚Äî file hash unchanged since last review |

Progress is validated with Zod (`ProgressSchema`) on load. Corrupted files are treated as empty.

## RAG Index

**Location:** `.anatoly/rag/`

The semantic vector index used by the duplication axis. See [RAG Semantic Index](rag-semantic-index.md) for full details.

| Path | Purpose |
|------|---------|
| `rag/lancedb/` | LanceDB database (function card embeddings, Parquet format) |
| `rag/cache.json` | Maps function ID (16-char hex) ‚Üí file SHA-256 hash |

**Incremental:** Only modified files are re-embedded. The cache tracks which functions have been indexed for which file hash.

**Auto-rebuild:** If the stored vector dimension doesn't match the current embedding model (768), the table is dropped and `cache.json` is cleared, forcing a full re-index.

## Lock File

**Location:** `.anatoly/anatoly.lock`

**Source:** `src/utils/lock.ts`

Prevents concurrent `anatoly run` instances from corrupting shared state.

```json
{
  "pid": 12345,
  "started_at": "2026-03-15T14:23:45.000Z"
}
```

### Behavior

| Function | Purpose |
|----------|---------|
| `acquireLock()` | Write lock at run start. Throws `LOCK_EXISTS` if another process holds it |
| `releaseLock()` | Delete lock on run completion or interruption |
| `isLockActive()` | Read-only check (used by hook/watch mode to detect active audit) |

**Stale lock detection:** If the PID in the lock file is no longer running, the lock is automatically removed and re-acquired.

**Manual recovery:** `anatoly reset` clears the lock file.

## Correction Memory

**Location:** `.anatoly/correction-memory.json`

Persistent storage for false positive corrections. When a reviewer marks a finding as incorrect, the correction is recorded here to prevent the same issue from being re-flagged on subsequent runs.

```json
{
  "version": 1,
  "false_positives": [...]
}
```

Persists across runs. Not cleared by `clean-runs` ‚Äî only by `reset`.

## Lifecycle Commands

### `anatoly reset`

Deletes **everything** inside `.anatoly/`:

| Target | Description |
|--------|-------------|
| `tasks/` | AST scan cache |
| `reviews/` | Legacy review directory |
| `logs/` | Legacy logs directory |
| `cache/` | Progress tracking |
| `runs/` | All run directories |
| `rag/` | Vector index + cache (drops LanceDB table via API first) |
| `progress.json` | Legacy progress file |
| `report.md` | Legacy report file |
| `anatoly.lock` | Lock file |

Requires confirmation (or `--yes` for CI).

### `anatoly clean-runs`

Deletes old runs from `.anatoly/runs/`:

```bash
anatoly clean-runs              # Delete ALL runs (with confirmation)
anatoly clean-runs --keep 3     # Keep 3 most recent
anatoly clean-runs --keep 1 -y  # Keep latest, no confirmation
```

Uses `purgeRuns()` which sorts runs chronologically (timestamp-based IDs sort naturally) and deletes the oldest first.

Also handles legacy: removes `.anatoly/logs/` if it exists and no runs/ directory is found.

## Atomic Writes

**Source:** `src/utils/cache.ts` ‚Äî `atomicWriteJson()`

All JSON state files use an atomic write pattern to prevent corruption on crashes or concurrent writes:

1. Write to a temp file (`.tmp-<timestamp>-<random>`)
2. Rename temp file to target path (atomic on most filesystems)
3. Clean up temp file in `finally` block

Used for: `progress.json`, `cache.json`, `run-metrics.json`, `correction-memory.json`.

## File Naming Convention

Source file paths are converted to flat output names by `toOutputName()`:

```
src/utils/format.ts  ‚Üí  src-utils-format
```

Rules:
1. Strip file extension
2. Replace `/` and `\` with `-`

This convention is used consistently for:
- `.task.json` files in `tasks/`
- `.rev.json` and `.rev.md` files in `runs/<runId>/reviews/`
- `.log` files in `runs/<runId>/logs/`
