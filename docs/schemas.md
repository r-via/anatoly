## ReviewFile

**File:** `src/schemas/review.ts` — The core output of each file evaluation.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `version` | `1 \| 2` | — | Schema version (v2 adds best_practices and axis_meta) |
| `file` | string | — | Reviewed file path |
| `is_generated` | boolean | `false` | `true` for triage-skipped files (synthetic review) |
| `skip_reason` | string? | — | Why the file was skipped (barrel, trivial, type-only, constants-only) |
| `verdict` | Verdict | — | `CLEAN` · `NEEDS_REFACTOR` · `CRITICAL` |
| `symbols` | SymbolReview[] | — | Per-symbol results from all 5 symbol-level axes |
| `actions` | Action[] | `[]` | Actionable recommendations (from correction axis) |
| `file_level` | FileLevel | — | File-level metadata (imports, deps, notes) |
| `best_practices` | BestPractices? | — | File-level best practices evaluation (v2 only) |
| `axis_meta` | Record\<AxisId, AxisMetaEntry\>? | — | Per-axis cost/duration metadata (v2 only) |

### SymbolReview

Each symbol (function, class, type, etc.) receives verdicts from 5 independent axes:

| Field | Type | Constraint | Description |
|-------|------|------------|-------------|
| `name` | string | — | Symbol name |
| `kind` | SymbolKind | — | `function` · `class` · `method` · `type` · `constant` · `variable` · `enum` · `hook` |
| `exported` | boolean | — | Whether the symbol is exported |
| `line_start` | integer | >= 1 | Starting line number |
| `line_end` | integer | >= 1 | Ending line number |
| `correction` | enum | — | `OK` · `NEEDS_FIX` · `ERROR` |
| `overengineering` | enum | — | `LEAN` · `OVER` · `ACCEPTABLE` |
| `utility` | enum | — | `USED` · `DEAD` · `LOW_VALUE` |
| `duplication` | enum | — | `UNIQUE` · `DUPLICATE` |
| `tests` | enum | — | `GOOD` · `WEAK` · `NONE` |
| `confidence` | integer | 0–100 | Confidence score |
| `detail` | string | min 10 chars | Explanation of the verdict |
| `duplicate_target` | DuplicateTarget? | — | Target info when `duplication = DUPLICATE` |

**DuplicateTarget:**

| Field | Type | Description |
|-------|------|-------------|
| `file` | string | Path to the duplicate file |
| `symbol` | string | Name of the duplicate symbol |
| `similarity` | string | Similarity description (e.g., "95% identical logic") |

### Action

Generated by the correction axis for `NEEDS_FIX` and `ERROR` findings:

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `id` | integer | — | Sequential ID (>= 1, assigned at merge) |
| `description` | string | — | Fix description (min 1 char) |
| `severity` | enum | — | `high` · `medium` · `low` |
| `effort` | enum | `small` | `trivial` · `small` · `large` |
| `category` | enum | `refactor` | `quickwin` · `refactor` · `hygiene` |
| `source` | AxisId? | — | Which axis produced this action |
| `target_symbol` | string \| null | — | Symbol name if applicable |
| `target_lines` | string \| null | — | Line reference (e.g., "L5-L10") |

### BestPractices

File-level evaluation against 17 TypeGuard v2 rules:

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `score` | number | — | 0.0–10.0 (start at 10, subtract penalties) |
| `rules` | BestPracticesRule[] | — | Array of 17 rule evaluations |
| `suggestions` | BestPracticesSuggestion[] | `[]` | Code improvement suggestions |

**BestPracticesRule:**

| Field | Type | Description |
|-------|------|-------------|
| `rule_id` | integer | 1–17 |
| `rule_name` | string | Rule name (e.g., "No `any`") |
| `status` | enum | `PASS` · `WARN` · `FAIL` |
| `severity` | enum | `CRITIQUE` · `HAUTE` · `MOYENNE` |
| `detail` | string? | Explanation of findings |
| `lines` | string? | Affected line numbers |

**BestPracticesSuggestion:**

| Field | Type | Description |
|-------|------|-------------|
| `description` | string | What to fix |
| `before` | string? | Current code example |
| `after` | string? | Corrected code example |

### FileLevel

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `unused_imports` | string[] | `[]` | List of unused import names |
| `circular_dependencies` | string[] | `[]` | List of circular dependency paths |
| `general_notes` | string | `""` | General observations about the file |

### AxisMeta

Per-axis performance metadata (v2 only):

| Field | Type | Description |
|-------|------|-------------|
| `model` | string | Model used for this axis |
| `cost_usd` | number | Cost in USD (>= 0) |
| `duration_ms` | number | Duration in milliseconds (>= 0) |

## Config

**File:** `src/schemas/config.ts` — Validates `.anatoly.yml` configuration.

| Section | Type | Default | Description |
|---------|------|---------|-------------|
| `project` | ProjectConfig | `{ monorepo: false }` | Project metadata |
| `scan` | ScanConfig | `{ include: ["src/**/*.ts", "src/**/*.tsx"], exclude: [...] }` | File discovery globs |
| `coverage` | CoverageConfig | `{ enabled: true, ... }` | Coverage integration |
| `llm` | LlmConfig | See below | LLM and axis configuration |
| `rag` | RagConfig | `{ enabled: true }` | Semantic RAG index |
| `badge` | BadgeConfig | `{ enabled: true, verdict: false, ... }` | README badge injection |
| `logging` | LoggingConfig | `{ level: "warn", pretty: true }` | Structured logging |
| `output` | OutputConfig | `{}` | Output management |

See [Configuration](configuration.md) for detailed field reference with examples.

### LlmConfig

| Field | Type | Default | Constraint | Description |
|-------|------|---------|------------|-------------|
| `model` | string | `"claude-sonnet-4-6"` | — | Primary model for reviews |
| `index_model` | string | `"claude-haiku-4-5-20251001"` | — | Model for RAG card generation |
| `fast_model` | string? | — | — | Cheaper model for fast-tier reviews |
| `agentic_tools` | boolean | `true` | — | Enable agentic tool use |
| `timeout_per_file` | integer | `600` | >= 1 | Seconds per file evaluation |
| `max_retries` | integer | `3` | 1–10 | Max retry attempts |
| `concurrency` | integer | `4` | 1–10 | Parallel reviews |
| `min_confidence` | integer | `70` | 0–100 | Min confidence for hook findings |
| `max_stop_iterations` | integer | `3` | 1–10 | Anti-loop limit for stop hook |
| `deliberation` | boolean | `false` | — | Enable Opus deliberation |
| `deliberation_model` | string | `"claude-opus-4-6"` | — | Deliberation model |
| `axes` | AxesConfig | all enabled | — | Per-axis configuration |

### AxesConfig

Each axis has:

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `enabled` | boolean | `true` | Enable/disable this axis |
| `model` | string? | — | Override model for this axis |

Axes: `utility`, `duplication`, `correction`, `overengineering`, `tests`, `best_practices`.

## Task

**File:** `src/schemas/task.ts` — AST scan output per source file. Written to `.anatoly/tasks/<file>.task.json`.

| Field | Type | Constraint | Description |
|-------|------|------------|-------------|
| `version` | `1` | literal | Schema version |
| `file` | string | — | Source file path |
| `hash` | string | — | SHA-256 content hash |
| `symbols` | SymbolInfo[] | — | Extracted symbols |
| `coverage` | CoverageData? | — | Optional coverage metrics |
| `scanned_at` | string | — | ISO 8601 scan timestamp |

**SymbolInfo:**

| Field | Type | Constraint | Description |
|-------|------|------------|-------------|
| `name` | string | — | Symbol name |
| `kind` | SymbolKind | — | `function` · `class` · `method` · `type` · `constant` · `variable` · `enum` · `hook` |
| `exported` | boolean | — | Whether the symbol is exported |
| `line_start` | integer | >= 1 | Starting line |
| `line_end` | integer | >= 1 | Ending line |

**CoverageData:**

| Field | Type | Constraint | Description |
|-------|------|------------|-------------|
| `statements_total` | integer | >= 0 | Total statements |
| `statements_covered` | integer | >= 0 | Covered statements |
| `branches_total` | integer | >= 0 | Total branches |
| `branches_covered` | integer | >= 0 | Covered branches |
| `functions_total` | integer | >= 0 | Total functions |
| `functions_covered` | integer | >= 0 | Covered functions |
| `lines_total` | integer | >= 0 | Total lines |
| `lines_covered` | integer | >= 0 | Covered lines |

## Progress

**File:** `src/schemas/progress.ts` — Pipeline state tracking. Written to `.anatoly/cache/progress.json`.

| Field | Type | Description |
|-------|------|-------------|
| `version` | `1` | Schema version |
| `started_at` | string | ISO 8601 timestamp of scan start |
| `files` | Record\<string, FileProgress\> | Per-file progress keyed by file path |

**FileProgress:**

| Field | Type | Description |
|-------|------|-------------|
| `file` | string | File path |
| `hash` | string | SHA-256 content hash |
| `status` | FileStatus | Current pipeline status |
| `updated_at` | string | ISO 8601 last update timestamp |
| `error` | string? | Error message (when status is `ERROR`) |

**FileStatus lifecycle:**

```
PENDING → IN_PROGRESS → DONE
                      → ERROR     (retryable on next run)
                      → TIMEOUT   (retryable on next run)
       → CACHED                   (hash unchanged, skip review)
```
